§ Warring Kingdoms


Overview

You are a ruler of a kingdom during the heart of the middle ages. To your dismay, a rival kingdom has popped up surprisingly close to yours. You must prove to the world that your kingdom is the greatest!

There are two ways to win Warring Kingdoms. The kingdom with the most collected food at the end of 500 turns wins. However, you can also win by razing the opponent's keep.

How to play (as a human)

Movement and attacking:

• Click on the unit you wish to move. The unit should become highlighted by a red cursor.
• Next click on a target.
• Then click on the move button located on the control panel at the right side of the screen.
• If you wish to attack the target, click on the attack button instead.

Farming:

• Farming is done by your farmer automatically.
• If your farmer dies, hopefully you have enough food to train another!

Training units:

• If you have enough money to train a unit, the unit's icon will appear on the control panel.
• Click this icon to train the unit. It will appear next to your keep.

Building:

• To build a wall, first click on the unit you want to use.
• Then click the target location. This must be directly adjecent to your unit.
• Finally, click on the build icon on the control panel.


The game state

Like all games at AI Tourney, the game state is stored in JavaScript Object Notation (JSON). This is what the warring kingdoms JSON looks like:

{
 "id": (number), // The gameID
 "game": "warring",
 "players": [
  {
   "username": (string)
   "units": [...] // Array of player's units
   "food": (number) // The player's total food
   "errors": (number) // Illegal move counter
  },
  { ... } // Same object for player two
 ],
 "map": [[...]] // 17 x 17 array of map tile objects
 "currentPlayer": (0 or 1), // Whether it is player0's or player1's turn
 "turn": (number), // Current turn count
 "gameOver": (0 or 1), // Whether the game is over (1) or not (0)
 "winner": (-1 or 0 or 1), // The winner player (-1 for no winner yet)
 "error": (string), // If any error occured it should show up here
}

Here is the JSON for a unit:

{
 "x": (number), // (x,y) coordinate in map array
 "y": (number),
 "class": "farmer", // or "solder" or "archer"
 "hp": (number), // Remaining health
 "attack": (number), // Unit's attack strength
 "range": (number), // Unit's attack distance
 "hasFood": (true/false), // Whether unit has food or not
 "canMove": (true/false), // If unit can move that turn (used internally right now)
}

Note, for a position (x,y) in the map, the index would be map[y][x]. This is how JSON map tile objects appear in the game state:

{
 "type": "keep", // or "farm" or "grass", etc...
 "hp": (number), // Remaining health
 "solid": (true/false), // Whether tile is solid or not
 "style": (number), // Sprite style (purely for asthetics)
}

Your move

The move in warring kingdoms should be a JSON (formatted as a string). Your bot executable should print a JSON string to the command line to make a move.

There are four types of updates: "move", "attack, "train", and "build". Each unit can take one action per turn. Here is what the JSON should look like:

{
 "updates": [
  {
   "type": "move",
   "unit": (number), // The index of the unit in your player.units[] array
   "direction": "N", // Or "E", "S", "W"
  },
  {
   "type": "attack",
   "unit": (number), // The index of the unit in your player.units[] array
   "direction": "N", // Or "E", "S", "W"
  },
  {
   "type": "train",
   "class": (string), // "farmer", "soldier", "archer"
  },
  {
   "type": "build",
   "unit": (number), // The index of the building unit in your player.units[] array
   "direction": "N", // Or "E", "S", "W"
  },
  {...}, // Updates for other units...
 ]
}
